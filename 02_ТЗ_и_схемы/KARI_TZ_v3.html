Делегирование прав директора региона
i
Директор региона может передать часть своих прав доверенному сотруднику. Делегирование подтверждается электронной подписью (ЭП), все действия делегата фиксируются в системе.

Как работает делегирование

ШагДействиеТехнически
1. Выбор делегата Директор региона выбирает сотрудника из списка пользователей системы Выпадающий список зарегистрированных сотрудников
2. Определение прав Выбирает, какие именно права передаются (чекбоксы): установка лимитов, утверждение запросов, просмотр исполнителей Матрица прав с чекбоксами
3. Срок действия Устанавливает дату окончания делегирования (обязательно) Datepicker, максимум 90 дней
4. Подтверждение ЭП Директор региона подтверждает делегирование SMS-кодом (ПЭП) SMS на телефон директора региона
5. Уведомление делегату Делегат получает Push + Email с информацией о делегировании и сроках FCM + почтовый сервис
6. Работа делегата Все действия делегата отображаются в системе с меткой «Делегат: [имя]», директор получает автоуведомления Метка в аудит-логе, Push-уведомления директору
7. Отзыв/истечение Директор может отозвать делегирование в любой момент; по истечении срока — автоматически Кнопка «Отозвать» + Cron-задача

Запрос на увеличение лимита (директор магазина / подразделения)

  • Директор магазина нажимает «Запросить увеличение лимита» → вводит желаемую сумму и обоснование
  • Запрос уходит директору региона (Push + Email уведомление)
  • Директор региона утверждает/отклоняет в кабинете с комментарием
  • При утверждении — лимит изменяется автоматически, инициатор получает уведомление
  • История всех запросов сохраняется с отметками кто/когда утвердил

Модуль биржи заданий

Для исполнителя (мобильное приложение):

  • Лента заданий с фильтрами: город, категория, дата, сумма выплаты
  • Карточка задания: название, описание, адрес магазина, дата/время, сумма, требования
  • Карта с геолокацией — ближайшие задания к исполнителю
  • Кнопка «Взять задание» — резервирование задания за исполнителем
  • Push-уведомления о новых подходящих заданиях
  • История всех заданий исполнителя

Статусная модель задания:

Открыто → Взято → На проверке → Принято / Отклонено → Оплачено

Категории заданий:

Мерчандайзинг Промо-акции Курьер Уборка Грузчик

Для директора магазина (веб-кабинет):

  • Создание задания: название, описание, категория, дата/время, адрес, сумма, количество исполнителей
  • Создание задания из шаблона — выбрать готовый шаблон (Выкладка обуви, Инвентаризация, Уборка, Промо) → заполнить только дату и сумму
  • Просмотр откликов — список исполнителей, откликнувшихся на задание
  • Приёмка: обязательный просмотр фотоотчёта + геометки от исполнителя → кнопки «Принять» или «Отклонить с комментарием» активируются только после просмотра фото
  • Повторное приглашение конкретного исполнителя на новое задание
  • Рейтинговая система исполнителей (1–5 звёзд) — оценка после каждой приёмки
  • Чёрный список исполнителей для магазина
3.3

Модуль документооборота (ЭДО)

  • Автогенерация договора ГПХ при регистрации исполнителя — по шаблону KARI
  • Автогенерация акта выполненных работ при приёмке задания директором
  • Подписание всех документов через ПЭП (SMS-код) — без бумажного документооборота
  • Хранение подписанных документов в системе — 5 лет
  • Скачивание документов в формате PDF
  • Реестр всех документов с поиском по: исполнителю, дате, сумме, статусу
  • Автоматическое получение чека от самозанятого через API ФНС после каждой выплаты
i
ПЭП (Простая электронная подпись) — юридически значима согласно 63-ФЗ при наличии соглашения сторон о признании ЭП. Соглашение включается в договор ГПХ при регистрации.
3.4

Модуль выплат

Логика расчёта выплаты:

$
Формула: Итоговая выплата = Сумма задания + Компенсация налога 6%
Пример: Задание 1 000 руб. → Выплата исполнителю 1 060 руб. (60 руб. — компенсация НПД за счёт KARI)

Контроль лимита 2,4 млн руб./год:

  • Предупреждение HRD при достижении 80% лимита (1 920 000 руб.)
  • Блокировка выплат при достижении 100% лимита (2 400 000 руб.)
  • Уведомление самому исполнителю о приближении к лимиту

Виды выплат:

ВидОписаниеИнициатор
Единичная выплата Ручной запуск через кабинет бухгалтерии по одному исполнителю Бухгалтерия
Массовая выплата Загрузка реестра → автоматическая проверка → запуск → отчёт о результатах Бухгалтерия

Обязательные проверки перед каждой выплатой:

  • Автоматическая проверка статуса самозанятого через API ФНС непосредственно перед переводом
  • Если статус потерян → блокировка выплаты + уведомление HRD
  • Проверка лимита 2,4 млн руб./год

Сроки и способы выплат:

  • Исполнитель самостоятельно выбирает периодичность: ежедневно / еженедельно / по факту закрытия задания
  • Перевод на карту любого банка РФ через Совкомбанк API
  • Уведомление исполнителя: Push + SMS

Интеграция с бухгалтерией:

  • Автоматическая передача данных в ФНС и получение чека после каждой выплаты
  • Выгрузка реестров в форматах Excel и XML для 1С

Способы наполнения стоп-листа

  • Excel-импорт (массовый): HRD загружает файл с колонками «ИНН», «ФИО», «Дата увольнения» — система автоматически считает blocked_until = дата увольнения + 2 года
  • Ручное добавление: HRD вводит ИНН, ФИО, причину и (для бывшего сотрудника) дату увольнения через форму в кабинете
  • Интеграция с 1С (перспектива): автоматическая синхронизация при увольнении сотрудника через HR-систему KARI

Автоматизация через Celery Beat

  • Ежедневно в 01:00 МСК: задача expire_stop_list снимает все блокировки, у которых blocked_until < сегодня
  • Освобождённые исполнители автоматически получают доступ к заданиям без вмешательства HR
  • Операция атомарна: если задача упала — повтор через 5 минут (max 2 попытки)

Что IT поддержка KARI делает на этапе пилота

  • Настраивает серверную инфраструктуру (Linux-сервер в РФ) по инструкции из проекта
  • Разворачивает (деплоит) готовый код на сервер
  • Следит за мониторингом (Grafana) во время пилота
  • При технических инцидентах — устраняет по инструкции или эскалирует
  • Создаёт резервные копии ежедневно
