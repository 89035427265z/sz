Настройка MinIO (хранилище фотоотчётов)

ℹ️
MinIO — это аналог Amazon S3, но на вашем сервере. Хранит фотоотчёты исполнителей. Требование ТЗ: хранить 3 года.
# Скачиваем MinIO
wget https://dl.min.io/server/minio/release/linux-amd64/minio
chmod +x minio
sudo mv minio /usr/local/bin/

# Создаём директорию для данных
sudo mkdir -p /data/minio
sudo useradd -r minio-user
sudo chown minio-user:minio-user /data/minio

# Создаём systemd сервис
sudo nano /etc/systemd/system/minio.service
# Содержимое /etc/systemd/system/minio.service:
[Unit]
Description=MinIO Object Storage
After=network.target

[Service]
User=minio-user
Group=minio-user
Environment="MINIO_ROOT_USER=minioadmin"
Environment="MINIO_ROOT_PASSWORD=СЛОЖНЫЙ_MINIO_ПАРОЛЬ"
ExecStart=/usr/local/bin/minio server /data/minio --console-address ":9001"
Restart=always

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable minio && sudo systemctl start minio
# Веб-консоль MinIO: http://сервер:9001
# Логин: minioadmin / СЛОЖНЫЙ_MINIO_ПАРОЛЬ

Миграция базы данных (Alembic)

ℹ️
Alembic создаёт все таблицы в базе данных. Нужно запустить один раз при первом деплое.
# Убедитесь что виртуальное окружение активировано
source /opt/kari/venv/bin/activate

# Перейдите в папку бэкенда
cd /opt/kari/04_Разработка/backend

# Проверить текущее состояние миграций
alembic current
# Если выводит "None" — миграции ещё не применялись (это нормально)

# Применить все миграции — создать все таблицы
alembic upgrade head

# Проверить что миграция применена
alembic current
# Должен вывести: 0001 (head)

# Проверить таблицы в БД
psql -U kari -d kari_samozanyatye -h localhost -c "\dt"
# Должны появиться: users, sms_codes, tasks, task_photos, task_templates,
# payments, payment_registries, payment_registry_items, fns_receipts
✅
Если видите все 9 таблиц — база данных готова к работе!

Будущие обновления БД

# При обновлении системы — применяем новые миграции той же командой
git pull origin main
alembic upgrade head
# Alembic сам поймёт какие миграции ещё не применены и выполнит только их

Запуск бэкенда (FastAPI)

Тестовый запуск (проверка)

source /opt/kari/venv/bin/activate
cd /opt/kari/04_Разработка/backend
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# Если запустилось успешно, увидите:
# INFO: Application startup complete.
# Откройте в браузере: http://сервер:8000/docs

Systemd сервис (постоянный запуск)

sudo nano /etc/systemd/system/kari-backend.service
[Unit]
Description=KARI.Самозанятые Backend API
After=network.target postgresql.service redis.service

[Service]
User=www-data
WorkingDirectory=/opt/kari/04_Разработка/backend
EnvironmentFile=/opt/kari/04_Разработка/backend/.env
ExecStart=/opt/kari/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 4
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable kari-backend
sudo systemctl start kari-backend
sudo systemctl status kari-backend  # Должно быть: active (running)

# Просмотр логов
sudo journalctl -u kari-backend -f

Запуск фронтенда (React)

cd /opt/kari/04_Разработка/frontend
cp .env.example .env
# Отредактировать: VITE_API_URL=http://ВАШ_СЕРВЕР:8000/api/v1

# Установить зависимости
npm install

# Для разработки (с автообновлением)
npm run dev
# Откроется на http://localhost:3000

# Для продакшена — собираем статические файлы
npm run build
# Результат в папке dist/ — отдаём через Nginx

Nginx для проксирования

sudo apt install nginx -y
sudo nano /etc/nginx/sites-available/kari
server {
    listen 80;
    server_name ВАШ_ДОМЕН_ИЛИ_IP;

    # Фронтенд (React)
    location / {
        root /opt/kari/04_Разработка/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # API бэкенда
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Swagger документация
    location /docs {
        proxy_pass http://127.0.0.1:8000;
    }
}
sudo ln -s /etc/nginx/sites-available/kari /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx

Celery — фоновые задачи

ℹ️
Celery запускает автоматические задачи по расписанию: проверку чеков ФНС (07:00), проверку статусов самозанятых (06:00).

Celery Worker (обрабатывает задачи)

sudo nano /etc/systemd/system/kari-celery-worker.service
[Unit]
Description=KARI Celery Worker
After=network.target redis.service

[Service]
User=www-data
WorkingDirectory=/opt/kari/04_Разработка/backend
EnvironmentFile=/opt/kari/04_Разработка/backend/.env
ExecStart=/opt/kari/venv/bin/celery -A app.tasks.fns_tasks worker --loglevel=info --concurrency=4
Restart=always

[Install]
WantedBy=multi-user.target

Celery Beat (планировщик расписания)

sudo nano /etc/systemd/system/kari-celery-beat.service
[Unit]
Description=KARI Celery Beat Scheduler
After=network.target redis.service

[Service]
User=www-data
WorkingDirectory=/opt/kari/04_Разработка/backend
EnvironmentFile=/opt/kari/04_Разработка/backend/.env
ExecStart=/opt/kari/venv/bin/celery -A app.tasks.fns_tasks beat --loglevel=info
Restart=always

[Install]
WantedBy=multi-user.target
sudo systemctl daemon-reload
sudo systemctl enable kari-celery-worker kari-celery-beat
sudo systemctl start kari-celery-worker kari-celery-beat
sudo systemctl status kari-celery-worker  # active (running)

Первый запуск — создание директора региона

ℹ️
После запуска сервера нужно создать первого директора региона (Юрий). Это делается один раз через специальный API-эндпоинт.
1
Открыть Swagger UI
Перейти в браузере: http://сервер:8000/docs
2
Найти endpoint POST /api/v1/users/init
В разделе "Пользователи" → нажать на POST /users/init → Try it out
3
Заполнить и отправить
{
  "phone":     "+79001234567",
  "full_name": "Юрий Иванов",
  "secret":    "INIT_SECRET из файла .env"
}
4
Войти в систему
POST /api/v1/auth/send-code → ввести телефон → получить SMS → POST /api/v1/auth/verify-code → получить токен
✅
После создания директора — эндпоинт /init автоматически отключится. Повторно создать второго регионального директора через него нельзя.

Финальная проверка работоспособности

  • PostgreSQL запущен и доступен
    sudo systemctl status postgresql → active
  • Redis запущен
    redis-cli ping → PONG
  • MinIO запущен и доступен
    Открыть http://сервер:9001 — должна открыться консоль MinIO
  • Таблицы созданы
    alembic current → 20260310_0005 (head) — все 5 миграций применены
  • Backend API отвечает
    curl http://сервер:8000/health → {"status":"ok"}
  • Swagger UI открывается
    Открыть http://сервер:8000/docs — список всех API
  • Фронтенд открывается
    Открыть http://сервер:3000 — страница входа с логотипом KARI
  • Celery worker запущен
    sudo systemctl status kari-celery-worker → active
  • Первый директор создан
    POST /api/v1/users/init выполнен успешно
  • SMS-авторизация работает
    В DEBUG=True код приходит в ответе API (поле debug_code)
