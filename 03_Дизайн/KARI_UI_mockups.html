01
Экран приветствия
Первый экран, который видит новый пользователь. Задача — передать доверие, простоту и ключевые преимущества работы с KARI за несколько секунд.
Ключевые элементы
  • Логотип KARI на тёмном фоне — фирменный стиль
  • 3 преимущества с зелёными галочками — убеждение
  • CTA-кнопка «Начать работу» — основное действие
  • Ссылка входа для существующих пользователей
  • Тёмный градиентный фон — профессиональный вид
Для разработчиков
Анимация появления при первом запуске (fade-in, 600ms).
Проверка токена при старте: если есть → пропустить экран.
Экран: SplashScreen.tsx
Route: / → redirect на /home если авторизован

02
Ввод номера телефона
Первый шаг регистрации. Минимум трений — только номер телефона, без email и паролей. Встроенная клавиатура исключает переключение раскладки.
Ключевые элементы
  • Прогресс «1 из 4» — пользователь знает, сколько шагов осталось
  • Прогресс-бар 25% — визуальный прогресс
  • Поле с маской +7 (___) ___-__-__
  • Встроенная цифровая клавиатура — удобство
  • Согласие с условиями — юридическая защита
Для разработчиков
Маска ввода: +7 (___) ___-__-__
Автофокус на поле при открытии экрана.
SMS через SMSC.ru API или МТС Exolve.
Кнопка «Получить код» активна только при 10 цифрах.
POST /api/auth/send-sms → { phone }

03
Подтверждение кода из SMS
Второй шаг регистрации. Верификация номера телефона через SMS-код. Автоматический переход между полями ускоряет ввод.
Ключевые элементы
  • 4 отдельных поля для каждой цифры кода
  • Визуальная обратная связь — заполненные поля красные
  • Таймер 60 секунд до повторной отправки
  • Прогресс «2 из 4» и полоска 50%
  • Показ номера телефона для подтверждения
Для разработчиков
Автопереход к следующему полю после ввода цифры.
При вставке из буфера — разбить 4 цифры по полям.
Таймер 60 сек → ссылка «Повторить» становится активной.
POST /api/auth/verify-sms → { phone, code }
Ответ: JWT access_token + refresh_token

04
Загрузка документов
Третий шаг регистрации. OCR автоматически распознаёт данные паспорта и ИНН, минимизируя ручной ввод. Интеграция с ФНС подтверждает статус самозанятого.
Ключевые элементы
  • Зона загрузки фото — камера или галерея
  • Автозаполнение полей через OCR (Яндекс Vision)
  • Зелёные рамки полей — данные успешно распознаны
  • Зелёная плашка — статус самозанятого подтверждён ФНС
  • Прогресс «3 из 4» и полоска 75%
Для разработчиков
OCR через Яндекс Vision API (распознавание паспорта).
Проверка ИНН: GET /api/fns/check-inn?inn=771234567890
Хранение фото: MinIO S3-совместимое хранилище.
Шифрование документов: AES-256 в БД.
Фото паспорта НЕ хранится после распознавания (GDPR/152-ФЗ).

05
Главный экран
Центральный хаб приложения. Пользователь сразу видит свой баланс, новые задания и текущую работу. Всё важное — в одном месте.
Ключевые элементы
  • Баланс-карточка с тёмным градиентом — главная цифра
  • Кнопки «Вывести» и «История» для быстрого доступа
  • Горизонтальный скролл новых заданий
  • Карточка «В работе» с оранжевой меткой
  • Нижняя навигация из 4 пунктов
Для разработчиков
Баланс: GET /api/wallet/balance
Задания: GET /api/tasks?limit=2&sort=new
В работе: GET /api/tasks?status=in_progress
WebSocket для обновления баланса в реальном времени.
Pull-to-refresh для обновления списка заданий.

06
Биржа заданий
Каталог всех доступных заданий с фильтрацией по типу работы. Пользователь может переключиться между списком и картой для удобного поиска.
Ключевые элементы
  • Фильтры-таблетки по категориям работ
  • Переключатель список / карта
  • Карточки с тегом «СРОЧНО» и номером задания
  • Кнопка «Взять →» прямо в карточке
  • Бесконечная прокрутка (пагинация)
Для разработчиков
Фильтры через query params: GET /api/tasks?category=merch
Пагинация: ?page=1&limit=10 (бесконечная прокрутка).
Кэш ответа: 5 минут в Redis.
Карта: Яндекс.Карты JS API v3.
Инвалидация кэша при взятии задания другим пользователем.

07
Карточка задания
Детальная страница задания со всей необходимой информацией. Прозрачное отображение оплаты с учётом налога формирует доверие у исполнителя.
Ключевые элементы
  • Фото/превью магазина для идентификации места
  • Полный адрес, дата, время, длительность
  • Описание и требования к исполнителю
  • Разбивка оплаты: задание + налог = итого
  • Большая красная кнопка «Взять задание»
Для разработчиков
GET /api/tasks/:id — загрузка данных задания.
POST /api/tasks/:id/take — взять задание.
Проверка статуса перед взятием (задание может быть занято).
Если задание уже взято → показать toast «Уже недоступно».
Налог 6% рассчитывается на бэкенде, не на клиенте.

08
Сдача выполненного задания
Экран фиксации результата работы. Три обязательных фото и подтверждение геолокации обеспечивают качественную приёмку и защищают от споров.
Ключевые элементы
  • Жёлтый статус «В РАБОТЕ» с таймером
  • Сетка 3 фото — минимальный фотоотчёт
  • Индикатор прогресса загрузки фото
  • Мини-карта с синей точкой геолокации
  • Кнопка активируется только после 3 фото
Для разработчиков
Загрузка фото: POST /api/tasks/:id/photos (multipart).
Геолокация: navigator.geolocation.getCurrentPosition()
Проверка радиуса: расстояние от магазина не более 200м.
Кнопка «Отправить»: активна когда photos.length >= 3 && inRadius.
POST /api/tasks/:id/submit → { photoIds[], comment, geoLat, geoLon }

Профиль пользователя
Личный кабинет с персональной статистикой, доступом к документам и настройкам. Рейтинг 4.9 мотивирует к качественной работе.
Ключевые элементы
  • Аватар с инициалами — персонализация

Биометрический вход: Face ID / Touch ID через expo-local-authentication.

Accessibility — WCAG 2.1

Аудит цветовой схемы и рекомендации для мобильного приложения

🎨 Аудит цветового контраста

Пара цветов Контраст WCAG AA (4.5:1) WCAG AAA (7:1) Вывод

📋 Ключевые правила реализации

👆 Touch targets — мин. 44×44dp
Все кнопки-иконки (чат, уведомления, фильтры) должны иметь зону касания минимум 44×44dp. Использовать hitSlop если визуально меньше.
🔊 Screen Reader labels
Каждая иконка-кнопка: accessibilityLabel с описанием действия. Статусные бейджи: роль text + метка «Статус: в работе».
⚡ Reduce Motion
Анимации конфетти и slide-transitions отключать при системной настройке «Уменьшить движение». Использовать AccessibilityInfo.isReduceMotionEnabled().
🎨 Цвет ≠ единственный индикатор
Статусы всегда дублировать иконкой + текстом:
❌ Только зелёная точка
✅ 🟢 «Активно» | 🟡 «На проверке» | 🔴 «Отклонено»
